# Contributing

Thanks for helping build EarBud.

This project is early, so contributions should keep the core idea sharp: an ear-first AI assistant that helps people move through their day and communicate more effectively.

## How To Contribute

Good early contributions include:

- Product ideas
- User flows
- Prototype code
- Privacy and safety improvements
- Conversation coaching prompts
- Mobile audio experiments
- Documentation improvements

## Development Values

- Keep the user in control.
- Treat audio and transcripts as sensitive data.
- Prefer clear consent flows.
- Build small working prototypes before large abstractions.
- Make suggestions short enough to be useful in real time.
- Avoid features that encourage manipulation or deception.

## Documentation Style

When editing docs:

- Use clear, direct language.
- Prefer concrete examples.
- Avoid hype.
- Explain privacy tradeoffs.
- Keep Markdown readable in plain text.

## Pull Request Checklist

Before opening a pull request, check:

- The change matches the project vision.
- Privacy impact has been considered.
- New behavior is documented.
- Any prototype code has basic setup instructions.
- The change avoids storing sensitive data unnecessarily.
